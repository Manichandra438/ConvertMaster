## What AI learned that isn't in docs yet
- `src/lib/tools.js` is explicitly commented as single source of truth for tool metadata — don't duplicate tool lists anywhere.
- Styling split: Tailwind+`cn()` for tool-page bodies, inline `style={{}}` + CSS custom properties (`var(--bg)` etc.) for the VSCode-shell chrome (Layout/Sidebar/ToolCard). Mixing these up would be a tell that new code doesn't match existing style.
- Shared `ft-*` CSS classes in `index.css` are the reuse point for any new file-upload tool (dropzone, file list, progress bar, page grid).
- No test suite exists in the repo at all.
- CI requires `--legacy-peer-deps` (React 19 peer conflicts) — plain `npm ci` will fail.
- `triggerDownload`/`formatBytes`/`sleep` helpers are duplicated verbatim across `ImageConverterTool.jsx` and `PdfTool.jsx` (not extracted to `lib/`) — worth checking `ResizeCompressTool.jsx` too before any refactor.
